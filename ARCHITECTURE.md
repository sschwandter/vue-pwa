# Architecture & invariants

This file documents the **load-bearing decisions** in the template — the things
that look arbitrary but exist to make the app feel native on iOS. Most of them
are also commented at the point of use; this is the single place that lists them
so you can tell, before editing, whether you're about to break one.

> If you change something here, change it deliberately. Several of these have a
> matching guardrail (`npm run check-theme`) or a regenerate step — see notes.

---

## The single scroll container

The app scrolls inside one fixed element, **not** the window:

- `index.html` has `<div id="app">`, and `src/main.ts` mounts Vue into it.
- `src/style.css` pins it with `position: fixed; inset: 0; overflow-y: auto`
  and locks `<body>` with `overflow: hidden; overscroll-behavior: none`.
- `src/router.ts` saves/restores the scroll position by reading this element via
  `getScroller()` (`src/dom.ts`, which owns the `APP_SCROLL_ID = "app"` string).

**Don't:**
- add `overflow: auto`/`scroll` to inner elements (breaks scroll save/restore),
- switch `#app` to `vh`/`dvh` height units (the fixed `inset: 0` is what matches
  the real standalone viewport, including the area behind the home indicator),
- make `<body>` scrollable (brings back iOS rubber-band/overscroll).

## Theme color — one source, mirrored in CSS

`theme.js` is the source of truth for the surface (background) colors. It feeds:

- the web manifest `background_color` / `theme_color` (`vite.config.ts`),
- the injected `theme-color` `<meta>` tags (the `inject-theme-color` Vite plugin
  in `vite.config.ts`),
- the iOS splash generator (`scripts/generate-splash.mjs`).

CSS **cannot** import JS, so `src/style.css` `:root` mirrors the same hex values
in tokens. **If you change a surface color, change it in both `theme.js` and
`style.css`.** `npm run check-theme` (runs in `build`) fails the build if they
drift apart.

Related constraints:
- `theme.js` stays a `.js` file so the Node splash script can `import` it; its
  types live in `theme.d.ts`.
- The root/`body` background must equal `--surface` — iOS fills the safe-area
  regions with the root background, so a mismatch shows as a colored strip.

## Generated regions you must not hand-edit

- **`index.html` splash block** — everything between
  `<!-- apple-splash:start -->` and `<!-- apple-splash:end -->` is written by
  `npm run generate-pwa-splash`. Edit `theme.js`/`public/logo.svg` and rerun;
  don't edit the tags or remove the markers.
- **`theme-color` metas** — injected by Vite at build time, intentionally absent
  from `index.html`. Don't add them by hand.
- **PWA icons + their `<link>` tags** — generated by `@vite-pwa/assets-generator`
  from `public/logo.svg` (`pwa-assets.config.ts`), injected at build.

## Routing contract

- **Every route must define `meta.title`.** It's typed as required in
  `src/router.ts` and drives both `document.title` and the iOS-style back-button
  label, so a route without one won't type-check.
- New screens should be **lazy-imported** (`component: () => import(...)`) so each
  ships as its own chunk. `src/router.ts` also handles the "stale chunk after
  deploy" case with a one-shot reload guard — keep that if you keep code-splitting.
- Back navigation is custom (`App.vue` `goBack`) because a standalone iOS PWA has
  no system back gesture. It prefers real history and falls back to `/`.

## PWA update flow (a matched set)

These settings work together; changing one in isolation breaks the update prompt:

- `registerType: "prompt"` (`vite.config.ts`) — the new service worker waits
  instead of auto-reloading.
- `workbox.clientsClaim: true` — so accepting the prompt actually reloads the
  open page (without it, "Reload" silently does nothing — this was a real bug).
- `workbox.globIgnores: ["**/apple-splash/**"]` — keeps ~30 splash PNGs out of
  the offline precache.
- `src/pwa.ts` + `src/components/UpdateToast.vue` surface `needRefresh` and call
  `applyUpdate()`. `pwa.ts` also re-checks on foreground (the main trigger on iOS)
  and on an hourly poll (`VITE_SW_POLL_INTERVAL` overrides the interval in ms).

## iOS quirk fixes in global CSS (`src/style.css`)

Each of these fixes a specific platform behavior — they're not cosmetic:

- **16px font on `input`/`textarea`** — anything smaller triggers focus-zoom.
- **`touch-action` rules** — remove the 300ms tap delay / double-tap zoom while
  keeping pinch-zoom for accessibility.
- **`user-select: none`** everywhere except inputs — native-app feel.
- **`prefers-reduced-motion`** — collapses animations to ~instant.
- Safe-area insets (`env(safe-area-inset-*)`) on the nav bar and content padding
  — keep content clear of the notch / Dynamic Island / home indicator.

## File map

| Path | Role |
| --- | --- |
| `src/main.ts` | App bootstrap; mounts into `#app`; requests persistent storage |
| `src/router.ts` | Routes + `meta.title` contract + #app scroll restore + stale-chunk reload |
| `src/App.vue` | App shell: nav bar, custom back button, `<RouterView>`, update toast |
| `src/pwa.ts` | Service-worker registration + update state |
| `src/dom.ts` | Owns the `#app` scroll-container id |
| `src/style.css` | Global resets + theme tokens (mirror of `theme.js`) + iOS fixes |
| `theme.js` / `theme.d.ts` | Source of truth for surface colors |
| `vite.config.ts` | Vite + PWA plugin + theme-color injection |
| `scripts/generate-splash.mjs` | iOS launch screens → `index.html` |
| `scripts/check-theme.mjs` | Build-time guard that `theme.js` and `style.css` agree |
